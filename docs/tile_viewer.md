# NAV Tile Viewer - ESP32 Map Simulator

`tile_viewer.py` is a specialized viewer for NAV binary tiles, designed for ESP32 navigation simulation. It displays NAV format tiles in a 768x768 viewport showing a 3x3 tile grid at 256px per tile.

## Features

- **NAV Binary Format Support**: Reads NAV v1 format tiles with geometry data
- **Interactive Navigation**: Mouse drag panning and scroll wheel zooming
- **Feature Identification**: Right-click to identify features at cursor position
- **Basic Caching**: Simple viewport-based feature caching for performance
- **Grid Overlay**: Optional tile grid display with tile coordinates
- **Polygon Rendering**: Toggle between filled and outlined polygons
- **Real-time Statistics**: Shows tile loading and feature count information

---

## What Does This Script Do?

- Loads and renders NAV binary format tiles for ESP32 navigation simulation
- Displays a 3x3 tile viewport centered on specified coordinates
- Supports zoom levels 1-19 with smooth panning and zooming
- Provides interactive controls via mouse and keyboard
- Shows tile boundaries and loading statistics
- Identifies features on right-click with detailed information
- Converts RGB565 colors to RGB888 for display

**Note:** This script is specifically designed to preview tiles generated by `tile_generator.py` for the IceNav ESP32 navigation system.

## Directory Structure

The tile viewer expects the following directory structure:

```
NAV_DIRECTORY/
├── 14/                    # Zoom level 14
│   ├── 1234/             # Tile X coordinate
│   │   └── 5678.nav      # NAV binary tile file
│   ├── 1235/
│   │   └── 5678.nav
│   └── ...
├── 15/                    # Other zoom levels
│   └── ...
└── ...
```

### NAV Tile Format

- **`.nav` files**: NAV binary format with 22-byte header + feature data
- **Header**: 4-byte magic (NAV1) + 2-byte feature count + 16-byte bbox
- **Features**: Geometry type, RGB565 color, zoom priority, width, coordinates

## Usage

### Command Line Arguments

```sh
python tile_viewer.py NAV_DIRECTORY --lat LATITUDE --lon LONGITUDE [--zoom ZOOM_LEVEL]
```

**Required Arguments:**
- `nav_dir`: Directory containing NAV tiles
- `--lat`: Center latitude (decimal degrees)
- `--lon`: Center longitude (decimal degrees)

**Optional Arguments:**
- `--zoom`: Zoom level (default: 14, range: 1-19)

**Example:**
```sh
python tile_viewer.py ./nav_tiles --lat 42.5063 --lon 1.5218 --zoom 14
```

## NAV Format Details

### Geometry Types
- `GEOM_POINT` (1): Point features
- `GEOM_LINESTRING` (2): Line features  
- `GEOM_POLYGON` (3): Polygon features

### Color Encoding
- RGB565 format (16-bit): 5 red bits + 6 green bits + 5 blue bits
- Automatically converted to RGB888 for display

### Zoom Priority Encoding
- High 4 bits: Minimum zoom level (0-15)
- Low 4 bits: Rendering priority (0-15, scaled by 7)

## Performance Features

- **Viewport Caching**: Features cached per viewport to avoid re-reading tiles
- **Lazy Loading**: Only loads tiles visible in 3x3 viewport
- **Coordinate Conversion**: Efficient lat/lon to pixel conversion
- **Priority-based Rendering**: Features sorted by priority for correct layering

## Troubleshooting

### Common Issues

1. **"Directory not found"**
   - Check that the NAV directory path is correct
   - Ensure directory structure follows `zoom/x/y.nav` pattern

2. **"Invalid magic in file"**
   - Verify files are in NAV format (should start with "NAV1")
   - Check that files are not corrupted

3. **"pygame required"**
   - Install pygame: `pip install pygame`

4. **No tiles displayed**
   - Check that tiles exist for the specified coordinates and zoom level
   - Verify coordinate format (decimal degrees)

### Debug Mode

Enable debug logging by modifying the logging level:

```python
# In tile_viewer.py, change:
logging.basicConfig(level=logging.DEBUG, format='%(asctime)s - %(levelname)s - %(message)s')
```
